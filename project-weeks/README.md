## IA Focus During Project Weeks

During project weeks, the IA role shifts from homework tracking to project progress support.

Continue to track:

- Attendance
- Daily progress
- Students who need help
- Students who are falling behind

## Daily Check-In Questions

Ask students short questions during work time:

- What did you finish yesterday?
- What are you working on today?
- What is blocking you?
- Have you committed your code today?

## Warning Signs

Tell the lead instructor if a student:

- Has not committed code recently
- Does not have a working project plan
- Avoids check-ins
- Is stuck on the same issue for a long time
- Misses project work time
- Has no clear MVP progress

## How to Support Students

Help students by:

- Asking them to explain the bug
- Reading error messages with them
- Encouraging small next steps
- Reminding them to commit working code
- Helping them focus on MVP before stretch goals

Avoid taking over the keyboard unless the lead instructor has approved that approach.

## Project Feedback

Project feedback should be supportive and practical.

Focus on:

- What is working
- What requirement is missing
- What the next highest-priority step is
